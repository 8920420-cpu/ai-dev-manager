# Цветовая схема приложений

Эта палитра обязательна для новых фронтендов и при развитии существующих
приложений. Цвет модуля используется как основной брендовый акцент: primary
actions, активная навигация, focus/selection и ключевые индикаторы. Нейтральные
фоны, текст и семантические цвета ошибок/предупреждений не перекрашивать в цвет
модуля.

| Модуль | Цвет | Базовый токен | Смысл | Статус |
|---|---|---:|---|---|
| Продажи | Зелёный | `#16A34A` | Деньги, рост, успех | Сделано |
| ПС Чат | Синий | `#2563EB` | Общение, доверие | Сделано |
| Закупки | Оранжевый | `#EA580C` | Активность, заказ, снабжение | Запланировано |
| Склад | Янтарный / коричневый | `#D97706` | Коробки, паллеты, логистика | Запланировано |
| Зарплата и кадры | Фиолетовый | `#7C3AED` | Люди, документы, управление | Запланировано |

## Правила применения

- При старте нового фронтенда сначала определить модуль по этой таблице и
  завести его цвет как семантический токен `--primary`, а не размазывать hex по
  компонентам.
- Производные состояния (`hover`, `active`, `soft`, `border`) строить из той же
  цветовой шкалы с достаточным контрастом; базовый токен соответствует примерно
  оттенку `600` палитры Tailwind.
- Для «Продаж» и «ПС Чата» уже реализованный дизайн является источником истины.
  Если фактический production-токен отличается от указанного базового оттенка,
  сохранять существующий токен и зафиксировать уточнение в этой таблице.
- Цвет модуля не заменяет семантические `success`, `warning`, `danger` и `info`:
  их смысл и контраст должны оставаться одинаковыми во всех приложениях.
- Текст, иконки и элементы управления на цветном фоне должны соответствовать
  WCAG AA по контрасту.
