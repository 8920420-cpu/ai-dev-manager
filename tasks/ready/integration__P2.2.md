# P2.2 PROJECT-TASK-MONITOR-001 — E2E монитора задач проекта

## Description

E2E монитора задач проекта

## Scope

- сквозные fixtures и browser/API E2E монитора; product-код вне scope.

## Pre-coding brief (готовит оркестратор)

- Зафиксировать временную шкалу событий, два изолированных projectId, timezone и управляемые clock/network delays.
- Подготовить disposable test data и cleanup; запись в общую БД без отдельного подтверждения запрещена.

## Tasks

- Подготовить два проекта с задачами на разных этапах, включая активную, завершённую, заблокированную и задачу с повторным входом в этап.
- Открыть подраздел первого проекта и сверить строки, агрегаты, этапы и длительности с ответом API; затем переключиться на второй проект во время незавершённого запроса.
- Проверить рост длительности активного этапа, неизменность завершённых длительностей, отображение `null`, ручное и автоматическое обновление.
- Проверить отсутствие межпроектной утечки, прекращение polling после закрытия монитора и корректную работу пагинации/фильтров.

## Acceptance

- E2E доказывает поток БД/история событий → API статистики → таблица выбранного проекта без ручного преобразования данных.
- Этапы и длительности совпадают с зафиксированными временными границами, а быстрые переключения проектов не смешивают ответы.
- Тест стабилен при повторном запуске и не зависит от локального часового пояса браузера.

## Orchestrator validation

- Запустить backend/frontend unit suites и browser E2E дважды с разным порядком проектов.
- Проверить остановку polling по сетевым вызовам и отсутствие данных соседнего projectId в DOM/API.
