# P0.2 PIPELINE-STAGE-CONFIG-001 — монитор не должен показывать удалённые из этапов роли

## Description

После удаления роли из раздела «Этапы пайплайна» эта роль всё ещё отражается на мониторе проекта. Нужно синхронизировать монитор с сохранённой конфигурацией этапов, чтобы он не показывал роли, которые больше не назначены ни одному этапу текущего проекта.

## Scope

- `src/features/projects/ProjectMonitor.tsx`
- `src/features/projects/ProjectMonitor.test.tsx`
- При необходимости: связанные типы/API-маппинг проекта в `src/types/project.ts` и `src/api/projectsApi.ts`.

## Pre-coding brief (готовит оркестратор)

- Проверить, не использует ли монитор статический список ролей/этапов вместо `project.stages` и `project.roles`.
- Источником истины для отображения ролей на мониторе должна быть актуальная сохранённая конфигурация проекта.
- Исторические задачи в таблице монитора можно показывать с их сохранёнными `stageName`/`stageCode`, но агрегаты/панель ролей не должны воскрешать удалённые из пайплайна роли.

## Tasks

- Найти все места в мониторе, где роли или этапы собираются из констант, fallback-словарей или старого состояния после редактирования проекта.
- Перестроить список ролей/этапов монитора от актуальных `project.stages` и назначенных `roleIds`/`roleCodes`; удалённые назначения не должны попадать в панель.
- Обеспечить обновление открытого монитора после сохранения редактирования проекта без необходимости полной перезагрузки страницы.
- Добавить regression-тест: открыть проект, удалить роль из «Этапы пайплайна», сохранить, убедиться что монитор больше не показывает эту роль в агрегатах/панели.
- Добавить тест, что исторические строки задач при этом не ломаются и продолжают показывать свой `stageName`, если он пришёл из API задач.

## Acceptance

- Роль, удалённая из «Этапы пайплайна», после сохранения не отображается на мониторе проекта в списке/агрегатах ролей.
- Если монитор был открыт во время редактирования, после успешного сохранения он получает актуальный проект и перерисовывается.
- Статический `ROLE_PIPELINE` или аналогичный fallback не используется как источник видимых ролей текущего проекта.
- Исторические строки задач остаются читаемыми: их `stageName`/`stageCode` отображаются из данных API, но не добавляют удалённую роль обратно в конфигурационную панель.
- `npm test -- --run ProjectMonitor` и `npm run build` проходят.

## Orchestrator validation

- Запустить focused-тесты `ProjectMonitor`.
- Ручной smoke: удалить роль из этапа пайплайна, сохранить проект, открыть/обновить монитор и проверить отсутствие удалённой роли.
