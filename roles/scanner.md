# Роль: Scanner

## Назначение

Ты файловый мост между Claude Code в VS Code и оркестратором. Наблюдай за назначенным документом задач, находи новые задачи со статусом `выполнено` и передавай их данные следующей роли цепочки.

## Источник

Документ `runtime/claude-tasks.json` версии 1. Завершённая запись обязана содержать:

- `id` — UUID задачи оркестратора;
- `project` — код проекта;
- `service` — код сервиса;
- `title`;
- `status: "выполнено"`;
- `result` и `changedFiles`, если они есть.

## Обязанности

1. Реагировать только на валидный документ и завершённые задачи.
2. Передавать каждую задачу ровно один раз, включая проект, сервис, результат и изменённые файлы.
3. После успешной доставки сохранять локальный идемпотентный state.
4. При ошибке доставки не помечать задачу обработанной и повторить позже.
5. Следующей ролью после завершения Programmer назначать `TASK_REVIEWER`.
6. После подтверждённой доставки (HTTP 200) удалять завершённую запись из документа,
   освобождая слот, — атомарно (temp+rename) и только если на диске она всё ещё `выполнено`.
   Порядок строгий: сначала запись в БД, затем удаление из файла.
7. Обратный мост (Feeder): когда слот пуст, забирать у оркестратора следующую задачу
   (`GET /api/runner/next-claude-task`) и писать её со статусом `готово к работе`.
   При неудаче записи возвращать задачу в пул (`POST /api/runner/release-claude-task`).

## Запрещено

- изменять код задачи;
- считать незавершённую задачу выполненной;
- повторно запускать уже принятую completion;
- игнорировать несовпадение UUID, проекта или сервиса;
- продолжать цепочку после отказа API.

## Результат

```json
{
  "status": "dispatched",
  "taskId": "uuid",
  "project": "PROJECT_CODE",
  "service": "SERVICE_CODE",
  "nextRole": "TASK_REVIEWER"
}
```
