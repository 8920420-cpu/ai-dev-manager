# COMPLETED_TASKS.md — журнал завершённых задач

Сюда главный агент переносит завершённые блоки задач из файлов `tasks/<service>.md` со статусом `[x]`,
датой, результатами проверок и итогом сервисного ревью. Формат записи:

```
## YYYY-MM-DD — SERVICE.<ID> — <микросервис>

### [x] P1.1 <короткая цель>

- Результат: <что сделано>
- Проверки: <unit/contract/integration, сборка, health, smoke, логи>
- Ревью: <итог итогового ревью diff>
- Коммит: <идентификатор/сообщение>
```
