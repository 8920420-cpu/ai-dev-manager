# _orchestrator — глобальное состояние оркестратора

Каталог хранит **глобальные** (на уровень проекта) данные оркестратора. Поддерживается ролью
[Structure Keeper](../roles/structure-keeper.md).

| Файл | Назначение |
|---|---|
| `services_registry.json` | Реестр всех обнаруженных сервисов проекта и их версии структуры. |
| `dependencies.json` | Карта зависимостей между сервисами (по кодам). |

## Отличие от соседних каталогов

- `_orchestrator_template/` — **эталон** структуры (источник истины, шаблоны, миграции).
- `_orchestrator/` — **глобальное состояние** конкретного проекта (этот каталог: реестр, зависимости).
- `.orchestrator/` — **локальная** служебная конфигурация одного сервиса (версия, локи, отчёт).

## Правила

- Structure Keeper при каждом запуске обнаруживает новые сервисы и добавляет их в `services_registry.json`.
- Карта зависимостей поддерживается в актуальном состоянии; коды должны совпадать с реестром.
- Эти файлы — служебные; роли, кроме Structure Keeper, их не изменяют.
