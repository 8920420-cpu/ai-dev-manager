# Миграция 0.0.0 → 1.0.0 (bootstrap)

- **From:** `0.0.0` (структура отсутствует / сервис не инициализирован)
- **To:** `1.0.0`
- **Тип:** применяемая (bootstrap нового или legacy-сервиса)

## Мотивация

Привести произвольный проект/сервис к базовому эталону структуры 1.0.0: единая раскладка задач,
обязательная документация и служебная конфигурация оркестратора.

## Шаги (только каркас, без кода)

1. `ADD` каталог `tasks/` со стадиями: `inbox/`, `ready/`, `in_progress/`, `review/`, `qa/`,
   `blocked/`, `done/`, `archive/` (каждая с `.gitkeep`).
2. `ADD` шаблон `tasks/TASK.template.md` и `tasks/README.md` (из эталона).
3. `ADD` отсутствующие документы из `_orchestrator_template/docs/`: `PROJECT_MAP.md`,
   `ARCHITECTURE.md`, `API_MAP.md`, `DATABASE_MAP.md`, `DECISIONS.md`.
   Существующие документы не трогать.
4. `ADD` `.pipeline.json` из шаблона, если отсутствует.
5. `ADD` каталог `.orchestrator/` с `version.json`, `service.json`, `locks.json`, `last_scan.json`,
   `migrations.log`, `structure_report.md`.
6. `SET` `.orchestrator/version.json` → `{ "orchestrator_structure_version": "1.0.0" }`.
7. Зарегистрировать сервис в `_orchestrator/services_registry.json`, если его там нет.

## Идемпотентность

Все шаги создают только отсутствующие артефакты. Повторное применение к уже соответствующему
сервису не создаёт изменений.

## Критерий завершённости

Сервис содержит полный каркас эталона 1.0.0, `.orchestrator/version.json` = `1.0.0`,
сервис присутствует в реестре.
