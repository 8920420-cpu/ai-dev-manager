# Миграция 1.1.0 → 1.2.0 (ПРИМЕР формата)

> Пример будущей миграции. Не применяется, пока эталон остаётся на 1.0.0.

- **From:** `1.1.0`
- **To:** `1.2.0`
- **Тип:** пример

## Мотивация (пример)

Ввести обязательный документ `SECURITY_MAP.md` в комплект документации сервиса.

## Шаги (только каркас)

1. `ADD` шаблон `_orchestrator_template/docs/SECURITY_MAP.template.md`.
2. `ADD` отсутствующий `SECURITY_MAP.md` в каждый сервис из шаблона.
3. `ADD` `SECURITY_MAP.md` в список `docs` в `.orchestrator/service.json`.
4. `SET` `.orchestrator/version.json` → `1.2.0`.

## Критерий завершённости

Каждый сервис имеет `SECURITY_MAP.md`, версия структуры = `1.2.0`.
